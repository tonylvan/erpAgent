## 二、核心功能

### 2.1 工单生成

#### 来源一：预警中心自动生成

**触发条件**：
- 预警等级 ≥ 中风险
- 预警持续时长 > 阈值
- 预警类型匹配工单规则

**自动生成规则**：

```python
# 预警 → 工单映射规则
ALERT_TO_TICKET_RULES = {
    "cash_flow_warning": {
        "category": "财务风险",
        "priority_map": {
            "HIGH": "URGENT",
            "MEDIUM": "HIGH",
            "LOW": "MEDIUM"
        },
        "auto_assign": True,  # 自动派单
        "assign_rule": "finance_manager"  # 分配给财务经理
    },
    "inventory_warning": {
        "category": "库存风险",
        "priority_map": {...},
        "auto_assign": True,
        "assign_rule": "warehouse_manager"
    },
    "ar_overdue_warning": {
        "category": "应收账款",
        "priority_map": {...},
        "auto_assign": True,
        "assign_rule": "sales_manager"
    }
}
```

**工单标题模板**：
```
[预警转工单] {预警类型} - {预警等级} - {发生时间}
示例：[预警转工单] 现金流预警 - 高风险 - 2026-04-06 19:00
```

#### 来源二：智能问数一键生成

**用户操作流程**：
```
1. 用户在智能问数页面查询数据
   → "查询欠款最多的 Top 10 客户"

2. 系统返回结果并标注风险
   → 发现 3 个客户欠款超过信用额度

3. 用户点击"生成工单"
   → 系统自动填充工单内容

4. 确认并提交
   → 工单创建成功，自动分配处理人
```

**工单内容自动填充**：
```python
def create_ticket_from_query(query_result: dict) -> dict:
    """从查询结果生成工单"""
    return {
        "title": f"[数据异常] {query_result['query_type']} - 发现{query_result['risk_count']}个风险点",
        "description": f"""
## 查询内容
{query_result['original_query']}

## 发现的问题
{query_result['risk_summary']}

## 详细数据
{query_result['risk_details']}

## 建议操作
{query_result['recommendations']}
        """,
        "category": query_result['category'],  # 根据查询类型自动分类
        "priority": query_result['risk_level'],  # 根据风险等级自动设置优先级
        "source": "smart_query",
        "source_data": {
            "query_id": query_result['id'],
            "query_time": query_result['time'],
            "risk_items": query_result['risk_items']
        }
    }
```

#### 派单规则引擎

```python
# 部门处理人规则配置
ASSIGNMENT_RULES = {
    "财务风险": {

        "role": "finance_manager",

    },
    "库存风险": {
        "rule_type": "role",
        "role": "warehouse_manager",
        "backup_role": "operations_director"
    },
    "应收账款": {
        "rule_type": "sales_owner",  # 根据客户归属销售员
        "fallback_role": "sales_manager"
    },
    "IT 支持": {
        "rule_type": "round_robin",  # 轮询分配
        "team": "it_support_team"
    },
    "业务咨询": {
        "rule_type": "category_match",  # 根据业务分类匹配
        "category_field": "business_unit"
    }
}
```

#### 派单流程

```
工单创建
 │
 ├─→ 检查是否指定处理人
 │   ├─ 是 → 直接分配
 │   └─ 否 → 进入自动派单流程
 │
 ├─→ 根据工单分类匹配规则
 │   ├─ 财务风险 → 财务经理
 │   ├─ 库存风险 → 仓库经理
 │   ├─ 应收账款 → 对应销售员
 │   └─ IT 支持 → 轮询分配
 │
 ├─→ 检查处理人可用性
 │   ├─ 可用 → 分配成功
 │   └─ 不可用 → 使用备用处理人
 │
 └─→ 发送通知
     ├─ 站内消息
     ├─ 邮件通知
     └─ 企业微信/钉钉（可选）
```

### 2.3 工单处理

#### 工单状态流转

```
┌─────────────────────────────────────────────────────────┐
│ 工单状态流转图 │
├─────────────────────────────────────────────────────────┤
│ │
│ OPEN → IN_PROGRESS → RESOLVED → CLOSED │
│ │ ↑ │
│ │ └───────────────┘ │
│ │ 可重新打开 │
│ │ │
│ └─→ ESCALATED (升级) │
│ │ │
│ └─→ ON_HOLD (挂起) │
│ │
└─────────────────────────────────────────────────────────┘
```

#### 状态说明

| 状态 | 说明 | 触发动作 |
|------|------|---------|
| **OPEN** | 待处理 | 刚创建或重新打开 |
| **IN_PROGRESS** | 处理中 | 已分配处理人 |
| **RESOLVED** | 已解决 | 处理人标记完成，待确认 |
| **CLOSED** | 已关闭 | 创建人确认关闭 |
| **ESCALATED** | 已升级 | 超时或手动升级 |
| **ON_HOLD** | 已挂起 | 等待外部依赖 |

#### 处理操作

**处理人可执行操作**：

| 操作 | 说明 | 权限 |
|------|------|------|
| **接单** | 确认处理 | 所有处理人 |
| **转派** | 转给其他人 | 当前处理人 |
| **升级** | 上报给上级 | 所有处理人 |
| **挂起** | 暂停处理 | 需要填写原因 |
| **解决** | 标记已完成 | 需要填写解决方案 |
| **添加备注** | 内部沟通 | 所有处理人 |
| **@相关人员** | 请求协助 | 所有处理人 |

### 3.2 SLA 时效管理

**时效配置**：

| 优先级 | 响应时间 | 解决时间 | 超时升级 |
|--------|---------|---------|---------|
| **URGENT** | 30 分钟 | 4 小时 | 2 小时 |
| **HIGH** | 2 小时 | 24 小时 | 12 小时 |
| **MEDIUM** | 8 小时 | 3 天 | 24 小时 |
| **LOW** | 24 小时 | 7 天 | 3 天 |

**超时处理**：
```python
def check_sla_violations():
    """检查 SLA 违规则自动升级"""
    overdue_tickets = db.query(Ticket).filter(
        Ticket.status == "OPEN",
        Ticket.created_at < datetime.now() - timedelta(hours=Ticket.priority_sla)
    ).all()
    
    for ticket in overdue_tickets:
        # 自动升级
        ticket.status = "ESCALATED"
        # 通知上级
        notify_manager(ticket)
        # 记录违规则
        log_sla_violation(ticket)
```

class AssignmentEngine:
    """智能派单引擎"""
    
    def auto_assign(self, ticket: Ticket) -> str:
        """自动分配处理人"""
        # 1. 获取工单分类
        category = ticket.category
        
        # 2. 匹配派单规则
        rule = self.get_assignment_rule(category)
        
        # 3. 根据规则类型分配
        if rule.rule_type == "role":
            return self.assign_by_role(rule.role)
        elif rule.rule_type == "sales_owner":
            return self.assign_by_sales_owner(ticket.entity_id)
        elif rule.rule_type == "round_robin":
            return self.assign_round_robin(rule.team)
        
        return rule.fallback_role
    
    def get_assignment_rule(self, category: str) -> AssignmentRule:
        """获取派单规则"""
        pass

## 七、数据库设计

### 7.1 工单表

```sql
CREATE TABLE tickets (
    id SERIAL PRIMARY KEY,
    title VARCHAR(255) NOT NULL,
    status VARCHAR(20) DEFAULT 'OPEN',  -- OPEN/IN_PROGRESS/RESOLVED/CLOSED/ESCALATED/ON_HOLD
    priority VARCHAR(20) DEFAULT 'MEDIUM',  -- LOW/MEDIUM/HIGH/URGENT
    description TEXT,
    category VARCHAR(100),
    
    -- 人员
    created_by VARCHAR(100),
    assigned_to VARCHAR(100),
    resolved_by VARCHAR(100),
    
    -- 时间
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP,
    resolved_at TIMESTAMP,
    
    -- 来源追踪
    source VARCHAR(50),  -- alert_center/smart_query/manual
    source_id VARCHAR(100),  -- 来源记录 ID
    source_data JSONB,  -- 来源详细数据
    
    -- 关联实体
    entity_type VARCHAR(50),  -- Customer/Supplier/Order/Asset
    entity_id VARCHAR(100),  -- 实体 ID
    reference_id VARCHAR(100),  -- 关联单据号
    
    -- SLA
    sla_response_hours INTEGER,
    sla_resolution_hours INTEGER,
    first_response_at TIMESTAMP,
    due_at TIMESTAMP,
    
    -- 扩展字段
    tags VARCHAR(255)[],  -- 标签数组
    custom_fields JSONB  -- 自定义字段
);

-- 索引
CREATE INDEX idx_tickets_status ON tickets(status);
CREATE INDEX idx_tickets_priority ON tickets(priority);
CREATE INDEX idx_tickets_category ON tickets(category);
CREATE INDEX idx_tickets_assigned_to ON tickets(assigned_to);
CREATE INDEX idx_tickets_created_at ON tickets(created_at);
CREATE INDEX idx_tickets_source ON tickets(source, source_id);
CREATE INDEX idx_tickets_entity ON tickets(entity_type, entity_id);
```

### 7.2 工单评论表

```sql
CREATE TABLE ticket_comments (
    id SERIAL PRIMARY KEY,
    ticket_id INTEGER REFERENCES tickets(id),
    content TEXT NOT NULL,
    author VARCHAR(100),
    author_role VARCHAR(50),
    is_internal BOOLEAN DEFAULT true,  -- 内部评论
    attachments JSONB,  -- 附件列表
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP
);

CREATE INDEX idx_comments_ticket_id ON ticket_comments(ticket_id);
CREATE INDEX idx_comments_created_at ON ticket_comments(created_at);
```

### 7.3 工单操作日志表

```sql
CREATE TABLE ticket_logs (
    id SERIAL PRIMARY KEY,
    ticket_id INTEGER REFERENCES tickets(id),
    action VARCHAR(50),  -- created/assigned/transferred/escalated/resolved/closed/reopened
    old_value JSONB,  -- 变更前值
    new_value JSONB,  -- 变更后值
    operator VARCHAR(100),
    reason TEXT,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_logs_ticket_id ON ticket_logs(ticket_id);
CREATE INDEX idx_logs_action ON ticket_logs(action);
```
