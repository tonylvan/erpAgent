## ⚠️ 注意事项

### 安全性

⚠️ **开发环境仅限局域网使用**:
- ❌ 不要将服务暴露到公网
- ❌ 不要在公网环境使用开发服务器
- ✅ 生产环境使用 Nginx 反向代理

### 端口变化

⚠️ **端口可能变化**:
- 如果 5182 被占用，Vite 会自动使用 5183、5184 等
- 启动时查看终端输出的实际端口
- 使用 `netstat` 命令确认端口

### IP 地址变化

⚠️ **IP 地址可能变化**:
- DHCP 分配的 IP 可能变化
- 建议使用静态 IP 或路由器 IP 保留
- 每次启动前运行 `ipconfig` 确认 IP

## 🛠️ 故障排查

### 问题 1: 其他设备无法访问

**检查清单**:
1. ✅ 确认设备在同一局域网
2. ✅ 确认 IP 地址正确
3. ✅ 确认防火墙未阻止
4. ✅ 确认服务正在运行

**解决步骤**:
```bash
# 检查本机 IP
ipconfig | findstr IPv4

# 检查服务状态
netstat -ano | findstr :5182

# 测试防火墙（临时关闭）
Set-NetFirewallProfile -Profile Domain,Public,Private -Enabled False
```

### 问题 2: 端口被占用

**症状**: Vite 自动使用其他端口

**解决**:
```bash
# 查看占用端口的进程
netstat -ano | findstr :5182

# 终止进程
taskkill /F /PID <进程 ID>

# 或者使用新端口（Vite 会自动处理）
```

### 问题 3: CORS 错误

**症状**: 前端能访问但 API 调用失败

**解决**: 后端已配置 CORS 允许所有来源，检查后端是否正常运行：
```bash
curl http://192.168.1.113:8005/health
```
