## 🎯 概述

ERP Agent 使用**统一的配置管理系统**，所有配置通过环境变量和 `.env` 文件管理。

### 核心优势

✅ **集中管理** - 所有配置在一个文件中  
✅ **类型安全** - 使用 Python dataclass 提供类型检查  
✅ **环境隔离** - 支持 development/staging/production  
✅ **安全隔离** - 敏感信息不进入版本控制  
✅ **便捷访问** - 统一的配置加载接口

## ⚙️ 配置项详解

### 1. Neo4j 图数据库配置

```bash
# 连接配置
NEO4J_URI=bolt://127.0.0.1:7687
NEO4J_USER=neo4j
NEO4J_PASSWORD=Tony1985
NEO4J_DATABASE=neo4j

# 超时配置（秒）
NEO4J_TIMEOUT=20                    # 查询超时
NEO4J_CONNECTION_TIMEOUT=30         # 连接超时
NEO4J_MAX_CONNECTION_LIFETIME=3600  # 连接生命周期
NEO4J_MAX_CONNECTION_POOL_SIZE=50   # 连接池大小
```

**说明**:
- `NEO4J_URI`: Neo4j 连接地址（bolt 协议）
- `NEO4J_PASSWORD`: **必须修改为实际密码**
- `NEO4J_TIMEOUT`: 查询超时时间，建议 20-30 秒

---

### 2. PostgreSQL 数据库配置

```bash
# 连接配置
POSTGRES_HOST=localhost
POSTGRES_PORT=5432
POSTGRES_DB=erp
POSTGRES_USER=postgres
POSTGRES_PASSWORD=postgres

# 连接池配置
POSTGRES_POOL_SIZE=10
POSTGRES_MAX_OVERFLOW=20
POSTGRES_POOL_TIMEOUT=30
POSTGRES_POOL_RECYCLE=1800
```

**说明**:
- `POSTGRES_DB`: 数据库名称
- `POSTGRES_PASSWORD`: **必须修改为实际密码**
- `POSTGRES_POOL_SIZE`: 连接池大小，根据并发调整

---

### 3. Redis 缓存配置

```bash
# 连接配置
REDIS_HOST=localhost
REDIS_PORT=6379
REDIS_PASSWORD=          # 无密码留空
REDIS_DB=0

# 缓存过期时间（秒）
REDIS_DEFAULT_EXPIRE=3600
GSD_CACHE_EXPIRE=3600
```

**说明**:
- `REDIS_PASSWORD`: 如无需密码，留空即可
- `REDIS_DEFAULT_EXPIRE`: 默认缓存过期时间（1 小时）
- `GSD_CACHE_EXPIRE`: 智能问数缓存过期时间

---

### 4. FastAPI 后端服务配置

```bash
# 服务配置
BACKEND_HOST=0.0.0.0
BACKEND_PORT=8005
BACKEND_WORKERS=1
BACKEND_RELOAD=true

# CORS 配置（允许的前端地址）
CORS_ORIGINS=http://localhost:5173,http://localhost:5174,http://localhost:5175,http://localhost:5176
CORS_ALLOW_CREDENTIALS=true
```

**说明**:
- `BACKEND_PORT`: 后端 API 端口（当前使用 8005）
- `CORS_ORIGINS`: 允许的前端地址，逗号分隔
- `BACKEND_RELOAD`: 开发环境开启热重载

---

### 5. OpenClaw Gateway 配置

```bash
OPENCLAW_GATEWAY_URL=http://127.0.0.1:18789
OPENCLAW_GATEWAY_TOKEN=335bfe288d7b3d499d84d5b21d540ce21ff0c3e7dedbc18
OPENCLAW_GATEWAY_WS=ws://127.0.0.1:18789
```

**说明**:
- `OPENCLAW_GATEWAY_TOKEN`: **必须配置**，从 OpenClaw Dashboard 获取
- `OPENCLAW_GATEWAY_WS`: WebSocket 连接地址

---

### 6. GSD 智能问数配置

```bash
GSD_API_VERSION=v25
GSD_CACHE_ENABLED=true
GSD_MAX_HISTORY=100
GSD_QUICK_QUERY_THRESHOLD=5
```

**说明**:
- `GSD_API_VERSION`: API 版本（v25=模拟数据，v26=OpenClaw HTTP API）
- `GSD_CACHE_ENABLED`: 是否开启缓存
- `GSD_QUICK_QUERY_THRESHOLD`: 快捷查询自动添加阈值（点击次数）

## 💻 使用统一配置

### 方式 1: 直接导入配置对象

```python
from app.core.config import neo4j_config, postgres_config, redis_config

# Neo4j 配置
print(f"Neo4j URI: {neo4j_config.uri}")
print(f"Neo4j 密码：{'已配置' if neo4j_config.is_configured else '未配置'}")

# PostgreSQL 配置
print(f"PostgreSQL URL: {postgres_config.url}")

# Redis 配置
print(f"Redis 主机：{redis_config.host}:{redis_config.port}")
```

### 方式 2: 使用便捷函数

```python
from app.core.config import get_all_configs, print_config_summary

# 获取所有配置
configs = get_all_configs()
print(configs["neo4j"]["uri"])

# 打印配置摘要（启动时检查）
print_config_summary()
```

### 方式 3: 在 FastAPI 中使用

```python
# app/main.py
from fastapi import FastAPI
from app.core.config import (
    backend_config, 
    neo4j_config, 
    postgres_config,
    print_config_summary
)

app = FastAPI()

@app.on_event("startup")
async def startup_event():
    """启动时打印配置摘要"""
    print_config_summary()
    
    # 验证关键配置
    if not neo4j_config.is_configured:
        logger.warning("Neo4j 密码未配置，将使用模拟数据")
    
    if not openclaw_config.is_configured:
        logger.warning("OpenClaw Token 未配置")
```

## 🔒 最佳实践

### 1. 安全配置

✅ **使用环境变量**
```bash
# .env
POSTGRES_PASSWORD=${DB_PASSWORD_FROM_SECRETS}
```

✅ **生产环境使用强密码**
```bash
# 生成强随机密码
python -c "import secrets; print(secrets.token_urlsafe(32))"
```

✅ **不要提交 .env 文件**
```bash
# .gitignore
.env
.env.local
.env.*.local
```

### 2. 环境隔离

**开发环境** (`.env.development`):
```bash
ENVIRONMENT=development
DEBUG=true
LOG_LEVEL=DEBUG
```

**生产环境** (`.env.production`):
```bash
ENVIRONMENT=production
DEBUG=false
LOG_LEVEL=WARNING
JWT_SECRET_KEY=production-secret-key-xyz123
```

### 3. 配置验证

**启动时验证关键配置**:
```python
from app.core.config import neo4j_config, postgres_config

def validate_configs():
    """验证配置是否正确"""
    errors = []
    
    if not neo4j_config.is_configured:
        errors.append("Neo4j 密码未配置")
    
    if not postgres_config.password:
        errors.append("PostgreSQL 密码未配置")
    
    if errors:
        raise ValueError(f"配置错误：\n" + "\n".join(errors))
    
    print("✅ 所有配置验证通过")
```

## 🔧 故障排查

### 问题 1: 配置未生效

**症状**: 修改 `.env` 后配置未更新

**原因**: Python 已缓存环境变量

**解决**:
```bash
# 重启 Python 进程
# 或在代码中重新加载
from dotenv import load_dotenv
load_dotenv(override=True)
```

---

### 问题 2: Neo4j 认证失败

**症状**: `Neo.ClientError.Security.Unauthorized`

**原因**: 密码配置错误

**解决**:
```bash
# 1. 检查 .env 文件
cat .env | grep NEO4J_PASSWORD

# 2. 验证配置
python -m app.core.config

# 3. 确认实际密码
# 查看项目中的 Python 文件
grep -r "NEO4J_PASSWORD" D:\erpAgent\backend\*.py
```

---

### 问题 3: PostgreSQL 连接失败

**症状**: `psycopg2.OperationalError: connection refused`

**原因**: 数据库未启动或配置错误

**解决**:
```bash
# 1. 检查 PostgreSQL 服务
Get-Service -Name postgresql*

# 2. 验证配置
python -c "from app.core.config import postgres_config; print(postgres_config.dsn)"

# 3. 测试连接
psql -h localhost -U postgres -d erp
```

---

### 问题 4: Redis 连接失败

**症状**: `redis.exceptions.ConnectionError`

**原因**: Redis 未启动或端口错误

**解决**:
```bash
# 1. 检查 Redis 服务
Get-Service -Name Redis

# 2. 验证配置
python -c "from app.core.config import redis_config; print(redis_config.url)"

# 3. 测试连接
redis-cli ping
```
